# Project Format

> CWK project structure specification.

**Version:** Draft 1.0

---

# Purpose

This document defines the on-disk structure of a CWK project.

It specifies every file managed by CWK, their purpose, and the responsibilities associated with them.

This document does **not** define how these files are read or written.

It only defines their format and meaning.

---

# Project Root

A CWK-enabled repository contains a hidden directory named:

```text
.cwk/
```

This directory identifies the repository as a CWK project.

The absence of this directory means the repository has not been initialized.

---

# Directory Structure

Version 1 defines the following structure.

```text
.cwk/

├── metadata.json
├── config.json
├── state.json
└── runtime/
```

Additional files may be introduced in future versions.

---

# metadata.json

## Purpose

Contains immutable information about the CWK project.

This file identifies the project and stores metadata required for compatibility.

Example:

```json
{
    "formatVersion": 1,
    "createdAt": "...",
    "cwkVersion": "1.0.0"
}
```

---

## Rules

This file should only change when:

- the project format changes;
- CWK performs a migration.

Normal synchronization must never modify this file.

---

# config.json

## Purpose

Stores user-defined configuration.

Examples include:

- synchronization interval;
- runtime adapter;
- timezone;
- Claude model;
- synchronization prompt.

Example:

```json
{
    "runtime": "github-actions",
    "interval": 5,
    "timezone": "Europe/Paris",
    "model": "haiku",
    "prompt": "."
}
```

---

## Rules

Configuration changes are initiated by the user.

Normal synchronization must never modify this file.

---

# state.json

## Purpose

Stores the mutable state of the project.

Example:

```json
{
    "lastSuccessfulPing": 1751741400000
}
```

---

## Rules

The engine owns this file.

It is updated only after successful synchronization.

The engine must never expose this timestamp directly to the user.

---

# runtime/

## Purpose

Contains files generated by the selected runtime adapter.

Version 1 currently supports:

```text
runtime/

github-actions/
```

Future adapters may create:

```text
runtime/

cron/

docker/

systemd/
```

The engine must never depend on runtime-specific files.

---

# Repository Files

CWK may also generate runtime files outside `.cwk/`.

Example:

```text
.github/workflows/cwk.yml
```

These files belong to runtime adapters.

They are **not** considered part of the Core project format.

---

# Versioning

Every CWK project has a format version.

Future versions of CWK may migrate existing projects.

Example:

```json
{
    "formatVersion": 2
}
```

Migration is outside the scope of Version 1.

---

# Project Detection

A repository is considered a CWK project when:

- `.cwk/` exists;
- `metadata.json` is valid;
- `formatVersion` is supported.

CWK must never rely only on directory existence.

---

# Mutability

| File | Mutable |
|------|----------|
| metadata.json | Rarely |
| config.json | User |
| state.json | Engine |
| runtime/* | Runtime Adapter |

---

# Ownership

Each file has a single owner.

| File | Owner |
|------|-------|
| metadata.json | Project Manager |
| config.json | Configuration Manager |
| state.json | Core Engine |
| runtime/* | Runtime Adapter |

No component should modify files it does not own.

---

# Forward Compatibility

Unknown files inside `.cwk/` must be ignored.

This allows future versions of CWK to extend the project format without breaking older installations.

---

# Architectural Rule

The `.cwk/` directory is the canonical representation of a CWK project.

Every component should treat it as the single source of truth for project state and configuration.
